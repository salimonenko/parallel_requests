АСИНХРОННОСТЬ + ОБЩАЯ ДИНАМИЧЕСКАЯ ПАМЯТЬ ДЛЯ НЕСКОЛЬКИХ СКРИПТОВ PHP

Функция parallel_requests() может запустить несколько скриптов (процессов) параллельно, АСИНХРОННО. Относительные пути к таким скриптам, а также POST-параметры (в виде массивов) задаются в качестве аргументов этой функции.
Для примера, эта функция вызывает скрипты, имеющие относительные пути '/TEST/parallel_requests/p_setter.php', '/TEST/parallel_requests/p_getter.php'. Они будут вызываться с POST-параметрами, соответственно: 
array('port' => 'PORT', 'host' => 'HOST'), 
array('param1' => 'value1', 'param2' => 'value2'),
array()

Т.е. параллельно (асинхронно) будут вызваны следующие URL:
/TEST/parallel_requests/socket_server.php?port=PORT&host=HOST
/TEST/parallel_requests/p_setter.php?param1=value1&param2=value2
/TEST/parallel_requests/p_getter.php

Вызовы (запуски) этих URL осуществляются при помоще сокетов PHP (аналог сокетов из языка С). 

ВАЖНО: скрипты p_setter.php и p_getter.php имеют возможность ОБМЕНА ДАННЫМИ через динамическую память сокет-сервера посредством сокетов. В частности, данные, записанные в память (в виде массива) сервера скриптом p_setter.php, позже получает скрипт p_getter.php. Это - аналог реализации технологии динамической памяти в PHP под названием shmop. БЕЗ использования обращений к жесткому диску (файлов, баз данных, сессий сервера и пр.). 

На экран выводится ТОЛЬКО ЧАСТЬ информации-логов о результатах работы скриптов. ВСЕ логи, записанные в режиме реального времени и для ВСЕХ скриптов, находятся в файле info.log.

***************************************     ПОРЯДОК РАБОТЫ     ***********************************

1. Запуск осуществляется путем запуска файла index.php. Там следует нажать кнопку "Сделать запрос". 
2. В результате пойдет AJAX-запрос к файлу parallel_requests.php. 
3. Он, в свою очередь, сделает POST-запросы к 3-м указанным скриптам посредством обычных интернет-сокетов (НЕ вебсокетов!). Затем в том файле запускается цикл, прослушивающий ответы этих сокетов.
3.1. Вначале запускается socket_server.php. В целях наглядности, там установлен таймаут, равный 1 сек., т.е. этот сервер будет работать только 1 секунду, после чего на экране и в файле-логе появятся результаты работы.
Сокет-сервер создает неблокирующий сокет, который начинает слушать выбранный порт (9999) на локальном хосте 127.0.0.1. 
Далее запускается бесконечный (в пределах заданного таймаута) цикл, в котором прослушивается созданный сокет, т.е. Обрабатываются существующие подключения (достаются из них все пришедшие от клиентов данные). Как только данные появляются, из них удаляются заголовок и пустая строка. Данные, присланные клиентами, ДОЛЖНЫ иметь следующий формат:
ИмяКлиента:Переменные:set     <- это для добавления данных в память сокет-сервера
или 
ИмяКлиента:Переменные:get 	  <- это для извлечения данных из памяти сокет-сервера

	3.1.1. Если от клиента пришли данные вида CLIENT1:param1=>value1;param2=>value2;1780662672.1109:set, 
	то сервер получает из них имя клиента (CLIENT1), метод (set) и саму строку данных (param1=>value1;param2=>value2;1780662672.1109). Затем имя и эта строка сохраняются в массиве, представляющем собой динамическую память сокет-сервера. Эта память может быть доступной ЛЮБОМУ сокет-клиенту, направившему сокет-запрос к данному сокет-серверу. 
	После чего сокет-сервер записывает в сокет сивмолы "OK\n". Именно их получит клиент, направивший сокет-запрос серверу.
	3.1.2. Если от клиента пришли данные вида CLIENT1:1780662672.1109:get, то сервер извлекает из этого массива строчку, соответствующую имени клиента (CLIENT1). При этом будет извлечена строчка, записанная в него раньше всех. Если такая строчка есть. 
	После чего сокет-сервер записывает сокет ответ (который будет направлен соответствующему сокет-клиенту). Если искомая строчка нашлась в массиве, именно она будет записана в сокет. Если нет - туда будут записаны символы "0\n" (т.е. ноль и перевод строки). 
	Если в результате запросов с параметрами set ли get массив изменился, то он выводится как на экран, так и в лог-файл.

3.2. Затем запускается p_setter.php. Этот скрипт в течение 1 секунды, в цикле, выполняет несколько сокет-запросов к сокет-серверу, используя скрипт socket_client.php (сокет-клиент). Именно этот скрипт направляет сокет-запросы к сокет-серверу и получает от него сокет-ответы. Этот скрипт направляет запросы с параметром set, для ЗАПИСИ данных в динамическую память (массив) сокет-сервера.

3.3. Затем запускается p_getter.php. Этот скрипт так же в течение 1 секунды, в цикле, выполняет несколько сокет-запросов к сокет-серверу, используя скрипт socket_client.php (сокет-клиент). Этот скрипт направляет запросы с параметром get, для ИЗВЛЕЧЕНИЯ данных из динамической памяти (массива) сокет-сервера.
В итоге, в результате добавления/извлечения данных из памяти (массива) сокет-сервера этот массив будет периодически меняться, что будет отображаться на экране и в лог-файле. 

Все скрипты (socket_server.php, p_setter.php, p_getter.php) работают независимо друг от друга, асинхронно. Обмениваясь друг с другом данными (в пределах заданых таймаутов времени их работы - 1 сек.).
